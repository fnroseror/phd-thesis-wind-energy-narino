# 04_RESULTS

## Purpose

This folder contains the structured outputs generated by the scientific workflow of the doctoral thesis:

**“Estudio de la velocidad de viento e inclusión de parámetros físicos para la predicción de energía eléctrica producida por fuentes eólicas”.**

It stores the result layer of the repository, separating generated outputs from source code, figures, and tables.

The organization follows the scientific logic of the thesis:

1. physical characterization of wind behavior,
2. deterministic model comparison for WPD,
3. probabilistic uncertainty assessment through PI90,
4. structural irregularity outputs based on FNRR,
5. regional energy integration and projection to 2028,
6. extended outputs required for traceability and reproducibility.

---

## Scientific scope

The doctoral thesis develops a hybrid physical–statistical framework for Wind Power Density (WPD) forecasting in Nariño, Colombia, using approximately 8 million hourly meteorological records from 2017–2022.

The result structure documented here is aligned with the thesis chapters and with the reported outputs related to:

- Weibull and Rayleigh comparison,
- temporal dependence analysis (ACF/PACF),
- spectral analysis (FFT),
- time–frequency analysis (Wavelet),
- deterministic forecasting metrics,
- Skill Score against persistence,
- calibrated PI90 uncertainty,
- FNRR structural irregularity,
- free and usable wind energy,
- and regional projection toward 2028.

- annual usable energy,

### `06_extended_results/`

- extended metrics,
- additional horizons,
- residual support,
- model-specific auxiliary outputs.

## Repository role

This folder acts as the main evidence layer of the repository.

Its purpose is to preserve traceability between:

- the written thesis,
- the executable methodology in `03_CODE/`,
- the visual outputs in `05_FIGURES/`,
- the tabular outputs in `06_TABLES/`,
- and the supplementary technical support in `07_APPENDICES_SUPPORT/`.

---

## Closure criterion

`04_RESULTS/` is considered complete when:

1. all major thesis result groups are represented,
2. each output can be traced back to a script in `03_CODE/`,
3. naming is consistent across chapters,
4. and the result structure supports external academic review and reproducibility.
